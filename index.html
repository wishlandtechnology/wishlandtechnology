WireWhat Step‚Äëby‚ÄëStep Workflow

1. Subscriber Request
A subscriber requests a live audio/video stream for a specific location or event via the app or web interface.
2. Publisher Notification
Nearby publishers (or registered broadcasters) receive a notification containing the requested location and details so they can respond.
3. Publisher Accepts & Starts Streaming
The publisher accepts the request and starts the live stream using OBS, mobile app, or hardware encoder ‚Äî sending the feed to WireWhat ingest endpoints.
4. AI Content Checks
Automated checks run in real-time: GPS verification, nudity detection, copyright detection, and crime/violence filters. Streams failing checks are flagged or queued for review.
5. Forward for Streaming
If checks pass, WireWhat transcodes and forwards the stream to delivery networks (HLS/CDN) and triggers any configured alert/webhook workflows.
6. Subscriber Receives Video
The subscriber receives the live stream in-app or via the embedded player; they can react, request replays, or report content. Telemetry and logs are recorded for auditing.
Note: The AI checks are configurable ‚Äî organizations can define strict or relaxed rules, enable human review, or integrate third-party safety services.
